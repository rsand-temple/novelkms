## Backend as shipped (so the frontend targets the real contract)

**User-facing — `ReviewSafetyResource @Path("/review")`** (active profile gated: 409
`profile_required` / 403 suspended; unknown handle 404; block-self 400; unblock idempotent):
- `GET /review/blocks` → list of `{handle, displayName, blockedAt}`
- `POST /review/blocks` body `{handle}` → block (idempotent)
- `DELETE /review/blocks/{handle}` → unblock
- `POST /review/reports` body `{targetType, targetId | targetHandle, reason, detail}`
  → `{id, status}`. `targetType` ∈ REQUEST | REVIEW | PROFILE (PROFILE resolves by
  `targetHandle` server-side; no bare USER target). `reason` ∈ SPAM | HARASSMENT | COPYRIGHT
  | HATE | EXPLICIT | OTHER. `detail` max 2000 chars. File-and-forget — reporter cannot list
  own reports in Phase 1.

**Admin — `AdminModerationResource @Path("/admin/moderation") @RolesAllowed(ADMIN)`:**
- `GET /admin/moderation/reports?status=&limit=` (status blank/ALL → all). Rows are
  `ContentReportView {id, reporterHandle, targetType, targetId, reason, detail, status,
  resolutionNote, createdAt, resolvedAt}` — reporter shown by handle, never user_id.
- `POST /admin/moderation/reports/{id}/resolve` body `{reason, note}`
- `POST /admin/moderation/reports/{id}/dismiss` body `{reason, note}`
- `POST /admin/moderation/requests/{id}/remove` body `{reason, note}` → sets request REMOVED
  + audit + auto-resolves that target's OPEN reports
- `POST /admin/moderation/reviews/{id}/remove` body `{reason, note}` → sets review REMOVED
  + audit + auto-resolve
- `POST /admin/moderation/profiles/{handle}/suspend` body `{reason, note}` → profile
  SUSPENDED + audit + auto-resolve
- `POST /admin/moderation/profiles/{handle}/reinstate` body `{reason, note}` → profile ACTIVE

Cross-user denial is 404 (never 403) everywhere. Everything on the wire is by handle.

## Confirmed design decisions (all accepted)

- **13 = all four surfaces + Blocked list on My Profile.**
- **Auto-resolve-on-remove = yes** (removing request/review or suspending profile
  auto-resolves that target's OPEN reports).
- Report targets REQUEST / REVIEW / PROFILE (PROFILE by handle→profile.id server-side; no
  bare USER target).
- Report reasons: SPAM, HARASSMENT, COPYRIGHT, HATE, EXPLICIT, OTHER.
- Report statuses: OPEN / RESOLVED / DISMISSED. File-and-forget.
- Everything by handle, never user_id on the wire. Cross-user denial 404 never 403.
- Participation gated on active profile (409 `profile_required` / 403 suspended).
