# Artifacts (Project Files)

Each project has an **Artifacts** area — a file and folder store for non-manuscript material: query letters, research PDFs, cover-art source images, character reference photos, and anything else you want to keep beside the book.

Artifact files and folders are entirely separate from your manuscript scenes and Codex entries. They do not appear in export or word counts.

## Finding Artifacts

In the [navigation tree](#help:manuscript.nav-tree), each project has an **Artifacts** section below the Codex. Clicking a folder opens the **Artifacts Explorer** in the center pane, showing that folder's contents in a detail view (Name, Type, Size, Modified).

Folders are shown in the nav tree; files appear only in the Explorer — the same two-pane model you know from Windows Explorer or macOS Finder.

## Uploading files

With a folder selected, use the **Upload** button in the Explorer toolbar, or drag files from your OS onto the Explorer pane. A drop-zone overlay confirms when NovelKMS is ready to receive files. Each file is limited to 50 MB; your per-account storage quota is shown in the usage bar at the bottom of the Explorer.

## Working with files and folders

| Action | How |
|---|---|
| Create a folder | **New folder** toolbar button |
| Rename | Right-click → **Rename** |
| Move | Right-click → **Move to…** (dialog), or drag a file row onto a folder row in the Explorer |
| Navigate up | **Up** toolbar button, or the breadcrumb |
| Download | Right-click → **Download** |
| Preview (images) | Double-click the file, or right-click → **Preview** |

## Trash and storage

Deleted artifact files and folders go to [Trash](#help:manuscript.trash) just like manuscript items. **Trashing does not free storage quota** — only purging does. Once purged, the file bytes are permanently removed.

## Editing text files

Plain-text files (`.txt`, `.md`, and similar) open in an in-place text editor in the Explorer. Other file types are download-only; images open in a read-only preview modal.
