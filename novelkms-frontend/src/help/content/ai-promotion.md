# Promoting Findings to the Codex

A useful review finding can be **promoted** into a [Codex](#help:codex.overview) entry, capturing it as durable project knowledge.

- If the finding suggests a category and title, those are used; otherwise it falls back safely to a Notes entry.
- Promotion creates the Codex entry and marks the finding `PROMOTED`, so it leaves your active queue but stays auditable.

Promotion is distinct from dismissing: a promoted finding was acted on, not waved away.
