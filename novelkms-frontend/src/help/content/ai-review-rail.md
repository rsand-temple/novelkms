# The AI Review Rail

The review rail is a collapsible panel beside the editor for running chapter and scene reviews and triaging the findings.

Each run creates a permanent review artifact; running again creates a new one, so you keep a history. Findings are triaged bug-tracker style across three tabs:

- **Active** — `OPEN` and `DEFERRED` findings (your working queue).
- **Resolved** — `DONE`, `DISMISSED`, and `PROMOTED` findings.
- **History** — the review runs themselves.

For each finding you can **Mark Done**, **Dismiss**, **Defer**, **Reopen**, copy the note, or [promote it into the Codex](#help:ai.promotion). When a finding includes anchor text, click it to [jump to the passage](#help:ai.anchor) in the editor.

The goal of the workflow is simple: empty the active queue.
