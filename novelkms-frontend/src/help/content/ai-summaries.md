# Chapter & Book Summaries

Summaries are **reader-facing synopses**, separate from [memory documents](#help:ai.memory).

- A **chapter summary** is one readable paragraph generated from the chapter's prose. One per chapter; regenerating overwrites it.
- A **book summary** is a synopsis of up to ~1000 words, built **entirely from the chapter summaries** in order — never the raw manuscript, which is too large to summarize reliably in one pass.

Right-click a book → **View chapter summaries…** to see them all with staleness chips, above the book-summary panel. If coverage is missing or stale when you generate the book summary, NovelKMS offers to fill the gaps first.

Both can be hand-edited; editing marks them as edited.
